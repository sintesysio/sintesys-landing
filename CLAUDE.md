## 1. O que é a Sintesys.io

A Sintesys.io é uma consultoria de transformação operacional com Inteligência Artificial para PMIs italianas (10-50 funcionários, faturamento €1,8M-€12M/ano). O fundador é **Lamberto Grinover**.

A empresa não vende "IA" em abstrato — vende **eficiência operacional** através de uma camada de inteligência que conecta Marketing, Vendas, Operações e Suporte.

| Atributo | Detalhe |
|---|---|
| Target | PMIs italianas, 10-49 dipendenti |
| Faturamento target | €1,8M - €12M/ano |
| Ticket | €38.500 (consultoria 90 dias) |
| Modelo | Setup Fee + Monthly Retainer (€1,5k-€3k/mês) |
| Diferencial | Não instala software; instala o "Sistema Operativo" da empresa |
| Mercado total | ~240.000-255.000 empresas na Itália |

### O Método (Roadmap 90 Giorni)

**Mês 1 — La Pulizia del Caos (Desorganização):** Audit de processos, mapeamento de fluxos (WhatsApp, Email, Excel), implementação de "Single Source of Truth" (CRM/ERP + Hub Operativo), centralização da comunicação.

**Mês 2 — Il Motore dell'Automazione (Trabalho Manual):** Automação back-office (OCR inteligente para faturas), agentes de assistência nível 1 (chatbot IA para FAQ e reservas), integração de workflows entre ferramentas.

**Mês 3 — Il Cervello del Business (Controle e Eficiência):** Dashboard de comando (BI em tempo real com KPI: CAC, LTV, Margem por Projeto), IA preditiva treinada nos dados da empresa, formação "AI-First Culture" (+40% eficiência).

### Proposta de Valor

| Dor do Cliente | Solução Sintesys | Resultado Tangível |
|---|---|---|
| Desorganização | Hub Central Operativo | Informação a 1 click |
| Falta de Dados | Dashboard BI com IA | Decisões baseadas em fatos |
| Sobrecarga | Agentes IA e Automação | -50%+ tarefas repetitivas |
| Falta de Eficiência | Otimização de Processos | Maior margem por funcionário |

### Pitch e Posicionamento

**Regra de ouro: Não vender IA. Vender Tranquilidade e Margem.**

> "Hoje você gasta 30% do seu tempo procurando informações e 40% fazendo tarefas que um algoritmo faria melhor. Sintesys.io não instala apenas software; instalamos o Sistema Operativo que permite à sua empresa crescer sem que você precise trabalhar mais horas."

Posicionamento-chave:
- **Atacar o Burnout**: "Trabalha 12 horas por dia mas a empresa não cresce. O problema não é a crise, é o caos operacional."
- **Vender Controle, não Automação**: "Não vendemos robôs. Construímos um painel de controle onde você vê tudo em tempo real."
- **Quebrar objeção de preço**: "O investimento se paga sozinho. Usamos o Transizione 5.0."
- **O Inimigo Incomum**: "Seu inimigo não é a concorrência. É o tempo que sua equipe perde copiando dados do Excel para o sistema."

## 2. Público-Alvo

O público-alvo são **titolari/imprenditori** (donos) de PMIs italianas com 10-50 funcionários. Sempre falar diretamente ao DONO, nunca ao time de TI ou marketing.

### Perfil Demográfico

| Atributo | Detalhe |
|---|---|
| Papel | Titolare / Imprenditore (decisor único) |
| Funcionários | 10-50 |
| Faturamento anual | €1,8M - €12M |
| Gestão | Centralizadora (todas as decisões passam pelo titular) |
| Cultura | Familiar, baseada na confiança pessoal |

36,4% se sentem sozinhos no papel. 41,7% já consideraram fechar por stress/burnout. Não existe middle management: o titular é estrategista, CFO e "bombeiro" ao mesmo tempo.

### Dores Principais

1. **Caos Operacional e Shadow AI**: Empresa cresceu em faturamento mas a estrutura não acompanhou. Funcionários usam ChatGPT escondido (Shadow AI), criando riscos de segurança sem governança.
2. **Falta de Liquidez**: 42% das PMIs têm dificuldade de acesso ao crédito bancário. 38% sofrem de problemas estruturais de liquidez.
3. **Turnover de Talentos**: Turnover médio de 28% ao ano em setores tecnológicos. PMIs não competem com salários de grandes empresas.
4. **Decisões por Instinto**: Apenas 26% das PMIs são consideradas digitalmente maduras. Dados espalhados entre ERP obsoletos, Excel e WhatsApp.

### Crenças Limitantes

| Crença | Em Italiano | Dado Real |
|---|---|---|
| "Sempre fizemos assim" | "Si è sempre fatto così" | Resistência brutal à mudança |
| "Tecnologia é cara e só para grandes" | "La tecnologia costa troppo ed è solo per i grandi" | 27% não digitaliza por medo dos custos; 60%+ ignora incentivos fiscais |
| "IA vai roubar empregos" | "L'IA ruberà il lavoro o mi farà perdere il controllo" | 50% da Itália mais preocupada que entusiasmada com IA |
| "Concorrentes são desleais" | "I concorrenti sono sleali" | Postura defensiva, foco em corte de custos |

### Medos Profundos

1. **Medo de Delegar (Perder o Controle)**: A promessa NÃO deve ser "tire as mãos do volante" mas "delegue à IA para ter um controle perfeito e preditivo".
2. **Fantasma da Sucessão**: 1 em 2 PMIs tem problemas de sucessão. Todo o know-how está na cabeça do titular, não em um sistema.
3. **Medo do Fracasso Público**: A cultura italiana estigmatiza o fracasso. Um investimento errado em tecnologia é uma vergonha pessoal.

### Setores-Chave

| Setor | % do Target | Dor Específica |
|---|---|---|
| Comércio atacadista e varejista | 38,1% | Controle de estoque e margens |
| Manufatura (Made in Italy) | 29,0% | Eficiência back-office e controle de custos |
| Outros Serviços | 20,1% | Margens altas, eficiência operacional |
| Construções | 12,8% | Desorganização pós-crescimento rápido |

### Palavras-Chave Emotivas (usar SEMPRE no conteúdo)

**Em italiano**: Marginalità, Controllo, Tranquillità, Flusso di Cassa, Efficienza, Governance, Ordine, Metodo, Sicurezza, Prevedibilità.

**Nunca usar**: Gíria de internet, linguagem técnica não explicada, tom informal, emoji excessivos.

**Tom**: De empresário para empresário. Como um editorial do Il Sole 24 Ore. Consultor sênior, não influencer.

## 5. Estrutura de Dados (Drizzle Schema)

### Tabela `users`
Autenticação OAuth com campo `role` (enum: `user` | `admin`). O campo role controla acesso ao painel admin.

### Tabela `leads`
Leads simples capturados pelo popup do Giornale e outras páginas. Campos: name, email, phone, sector, source, createdAt.

### Tabela `qualified_leads`
Leads qualificados do formulário Contattaci (multi-step). Campos expandidos: perfil (revenue, employees, sector), diagnóstico de dores (mainObstacle, dataLocation, cashFlowChallenge, delegationChallenge), maturidade tech (currentTools, usesAI, aiDetails, shadowAIConcern), urgência (priority, successionConcern, isDecisionMaker).

### Tabela `daily_editions`
Conteúdo editorial dinâmico do Giornale. Cada registro = 1 dia. Campos: editionDate, editionNumber, headline, editorial (3 parágrafos), imageCaption, 3 blocos de stats (número, sufixo, label, descrição, fonte), quote, CTA.

### Tabela `clients`
Clientes do módulo financeiro. Campos: name, company, email, phone, notes.

### Tabela `transactions`
Transações financeiras (entradas/saídas). Campos: clientId, type (entrada/saida), amount (centavos), description, category, date.

## 8. Estratégia de Marketing

### Arquitetura Dual-Path de Captação

**Caminho 1 — Landing Page (/):** Página de conversão direta. Formulário popup de 6 campos (nome, email, telefone, setor, faturamento, funcionários). CTA: "Prenota la Sessione Strategica". Sem a palavra "gratuito". Gera lead qualificado (tag "Qualificato" no Mailchimp).

**Caminho 2 — Il Giornale dell'IA (/giornale):** Conteúdo editorial dinâmico no formato de jornal italiano clássico. Edições diárias que mudam automaticamente (manchete, editorial, citação, estatísticas). Popup de lead magnet (Guida Transizione 5.0) aparece após 2 segundos. Gera lead simples (tag "lead" no Mailchimp).

### Funil de Conteúdo

O conteúdo NÃO vende. O conteúdo educa, informa e posiciona. A venda acontece no funil pago (BOFU) e no time comercial.

**Prioridades (em ordem):**
1. Atrair visitantes ao perfil — conteúdo tão relevante que o empresário clica para ver mais
2. Converter visitantes em seguidores — feed deve parecer um portal de notícias sério
3. Gerar saves e shares — métricas que alimentam o algoritmo
4. Capturar leads para newsletter — soft CTA no final ("Iscriviti alla newsletter — link in bio")

### Estratégia de Conteúdo Orgânico (Instagram)

**Conceito**: "Il Giornale dell'IA" — portal de notícias de IA para empresários italianos.

**Pilares editoriais:**
1. **Punte Calde dall'Italia** (TOFU): Notícias, leis, incentivos, movimentos de grandes empresas
2. **Passato vs Futuro** (MOFU): Contraste entre velhas revoluções e a revolução IA
3. **Curiosità e Dietro le Quinte** (MOFU): Fatos surpreendentes, mitos desfeitos

**Regras visuais:**
- Capa: FOTO EDITORIAL REAL de alta qualidade, nunca ilustrações stock
- Overlay: Gradiente escuro na parte inferior para legibilidade
- Título: GIGANTE, CAPS, BOLD, branco, na metade inferior
- Logo: Sintesys.io SEMPRE presente
- Slides internas: Fundo escuro, texto branco, focado em dados

**Frequência**: Mínimo 3 posts excelentes por semana (Seg/Qua/Sex), pode estender para diário.

### Estratégia de Ads Pagos (Meta + Google)

**Fase Atual (Meses 1-3): Distribuição de Conteúdo** — Todo investimento em ads foca em distribuir o conteúdo editorial para atrair visitantes ao perfil e converter em seguidores. NÃO vender. NÃO gerar leads de venda.

**Distribuição de Budget:**
- 70% Meta Ads / 30% Google Ads
- TOFU (50% Meta): Impulsionar posts de notícias quentes → Visitas ao perfil
- MOFU (30% Meta): Conteúdo "Passado vs IA" para retargeting → Engajamento
- BOFU (20% Meta): Conteúdo premium + Lead Magnet → Leads newsletter

**KPIs Primários (Fase Atual):**
- Custo por Seguidor (CPF) < €2,00
- Custo por Lead Newsletter (CPL) < €12,00

**Regra de Ouro**: Anúncios devem parecer conteúdo nativo do feed (manchete de jornal), NÃO anúncios tradicionais.

### Email Marketing

**Automações ativas no Mailchimp:**
1. Welcome Lead Simples (ID 7061): Guida Transizione 5.0 — acionada por tag "Lead"
2. Welcome Lead Qualificado (ID 7062): Agendamento de sessão estratégica 30min — acionada por tag "Qualificato"

**Réguas de nutrição documentadas (configurar no Mailchimp):**
- Lead Simples: 28 dias, 6 emails (educação → case study → urgência → oferta)
- Lead Qualificado: 14 dias, 4 emails (confirmação → análise → case study → follow-up)

**Templates por setor documentados:** 7 setores (Manifattura, Commercio ingrosso, Commercio dettaglio, Servizi professionali, Costruzioni, Logistica, Ristorazione).

## 9. Painel Admin (/admin)

O painel admin é protegido por `adminProcedure` (verifica `ctx.user.role === 'admin'`). Para promover um usuário a admin, atualizar o campo `role` diretamente no banco de dados.

### Módulos Implementados (Fase 1)

1. **Dashboard** (`/admin`): Cards de métricas (total leads, qualificados, taxa conversão, saldo), gráfico de leads/dia (30 dias), gráfico leads/setor (pie chart), gráfico fluxo financeiro mensal.

2. **Leads** (`/admin/leads`): Tabela de leads simples e qualificados com busca por nome/email/setor. Tabs separadas por tipo.

3. **Pipeline CRM** (`/admin/pipeline`): Kanban visual com 4 colunas (Lead, Qualificato, In Negoziazione, Chiuso). Integrado com Notion API real — lê deals do CRM Pipeline. Drawer lateral com detalhes completos do deal (metadados + conteúdo do Notion).

4. **Campanhas** (`/admin/campanhas`): Integrado com Mailchimp API real. Cards de métricas (subscribers, open rate, click rate, total campanhas). Tabela de campanhas com performance individual (recipients, opens, clicks, bounces, unsubscribes).

5. **Financeiro** (`/admin/financeiro`): CRUD de clientes, CRUD de transações (entrada/saída por cliente), panorâmica com saldo total e por cliente, filtro por período e por cliente.

### Pendências Fase 3

- Chat IA integrado com contexto dos dados da empresa
- Drag-and-drop no Kanban (mover deals entre colunas → atualizar status no Notion)
- Relatórios automatizados (semanal/mensal)

## 14. Decisões de Produto Tomadas

1. **"Gratuito" removido** de todos os CTAs — a palavra pode soar mal para o público-alvo (empresários). Substituído por "Sessione Strategica".
2. **Foto do Lamberto** apenas na seção bio no final da LP, não no hero.
3. **Popup reabre** após ser fechado (2s delay) — sem desistir permanentemente.

## 17. Como Conectar o Claude Code a Este Projeto

### Passo a Passo

**1. Exportar o código para GitHub**

No painel do Manus, vá em **Settings → GitHub** e exporte o repositório. Escolha um nome (ex: `sintesys-landing`) e o owner da sua conta GitHub.

**2. Clonar o repositório localmente**

```bash
git clone https://github.com/SEU_USUARIO/sintesys-landing.git
cd sintesys-landing
```

**3. Instalar dependências**

```bash
pnpm install
```

**4. Instalar o Claude Code (se ainda não tiver)**

```bash
npm install -g @anthropic-ai/claude-code
```

Ou via Homebrew no macOS:

```bash
brew install claude-code
```

**5. Iniciar o Claude Code na raiz do projeto**

```bash
claude
```

O Claude Code vai automaticamente detectar o arquivo `CLAUDE.md` na raiz e carregar todo o contexto do projeto. Ele terá acesso a:

- Todo o código-fonte (frontend + backend)
- Schema do banco de dados
- Testes existentes
- Documentação de negócio, estratégia e workflows
- Integrações (Mailchimp, Notion, Instagram)
- Decisões de produto e roadmap

**6. Configurar variáveis de ambiente (para rodar localmente)**

Crie um arquivo `.env` na raiz do projeto com as variáveis necessárias. As variáveis estão listadas na seção 13 deste documento. No ambiente Manus, elas são injetadas automaticamente.

```bash
# .env (exemplo — preencher com valores reais)
DATABASE_URL=mysql://user:pass@host:3306/db
JWT_SECRET=seu-jwt-secret
NOTION_API_KEY=ntn_xxx
MAILCHIMP_API_KEY=xxx-us14
MAILCHIMP_LIST_ID=b0d9ab0ecc
MAILCHIMP_SERVER_PREFIX=us14
```

**7. Comandos úteis no Claude Code**

Uma vez dentro do Claude Code, você pode pedir:

```
# Entender o projeto
```

### Configuração Avançada: MCP Servers

Para dar ao Claude Code acesso em tempo real ao Notion e Mailchimp (além do código), configure MCP servers no arquivo `.claude/mcp.json`:

```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "NOTION_API_KEY": "ntn_xxx"
      }
    }
  }
}
```

Isso permite que o Claude Code leia e escreva diretamente no Notion CRM, pipeline de conteúdo e tarefas.
